REPORTE TÉCNICO: VALIDACIÓN Y PRUEBAS DEL SISTEMA SECURITYEYE
============================================================

1. INTRODUCCIÓN
Este documento detalla la metodología y resultados de las pruebas de validación realizadas sobre el sistema SecurityEye. El objetivo principal fue verificar la estabilidad, rendimiento y funcionalidad del sistema bajo condiciones de carga concurrentes, simulando un escenario real de uso en un entorno educativo.

2. ENTORNO DE PRUEBAS
Las pruebas se ejecutaron sobre el despliegue de producción del sistema:
- Plataforma de Despliegue: Render Cloud (Nivel Gratuito).
- Base de Datos: PostgreSQL gestionada en la nube.
- Algoritmo de Seguridad: SHA-256 para hashing de credenciales (seleccionado por su eficiencia en entornos de recursos limitados).
- Herramientas de Prueba: 
    - Gemini CLI (Orquestación).
    - Python + Httpx + AsyncIO (Simulación de carga asíncrona).
    - Playwright (Validación de navegabilidad y flujo de usuario).

3. METODOLOGÍA

3.1 Pruebas de Estrés (Stress Testing)
Se diseñó un script automatizado en Python para evaluar la capacidad del backend de manejar múltiples solicitudes de registro simultáneas.
- Escenario: 30 usuarios intentando registrarse en el sistema en el mismo instante (t=0).
- Objetivo: Validar que el servidor puede procesar una clase completa de estudiantes sin colapsar ni generar tiempos de espera inaceptables.

3.2 Pruebas de Caja Negra (Black-Box Testing)
Se utilizó Playwright para simular la interacción de un usuario real con la interfaz gráfica, verificando que los flujos de registro e inicio de sesión funcionan correctamente desde el navegador, independientemente de la lógica interna del código.

4. RESULTADOS OBTENIDOS

4.1 Análisis de Escalabilidad (Pruebas de Estrés)
Se realizaron dos experimentos controlados para evaluar la respuesta del sistema ante diferentes niveles de demanda.

-----------------------------------------------------------------------
| Métrica                       | Escenario A (30u)  | Escenario B (100u) |
-----------------------------------------------------------------------
| Usuarios Concurrentes         | 30                 | 100                |
| Tasa de Éxito                 | 100% (30/30)       | 100% (100/100)     |
| Tiempo Total de Ejecución     | 3.49 segundos      | 12.71 segundos     |
| Tiempo Promedio por Registro  | 2.14 segundos      | 7.25 segundos      |
| Errores de Servidor (50x)     | 0                  | 0                  |
-----------------------------------------------------------------------

4.2 Discusión de Resultados
- Robustez: El sistema mantuvo una tasa de éxito del 100% incluso al triplicar la carga de usuarios (de 30 a 100).
- Escalabilidad Lineal: El incremento en el tiempo total de ejecución (de 3.49s a 12.71s) muestra un comportamiento predecible y estable de la arquitectura.
- Gestión de Recursos: A pesar de las limitaciones de la infraestructura gratuita de Render Cloud, la optimización mediante SHA-256 y la gestión asíncrona de FastAPI permitieron procesar la ráfaga de peticiones sin pérdida de datos.


4.2 Resultados Cualitativos
- La base de datos mantuvo la integridad de los datos sin generar duplicados ni bloqueos (deadlocks) durante la inserción masiva.
- El servidor no reportó errores de tipo 50x (Bad Gateway / Timeout), indicando que los recursos de CPU y memoria fueron gestionados correctamente.

5. CONCLUSIÓN
Las pruebas realizadas confirman que la arquitectura del sistema SecurityEye, optimizada con SHA-256, es apta para su despliegue en entornos reales con una concurrencia de hasta 30 usuarios. El sistema cumple con los requisitos de rendimiento y estabilidad necesarios para el artículo científico.
